# Generated namespaces

Auto-generated route bindings — one file per bot route file. Each export is an `attach<Namespace>(client)` function the `ModlyClient` constructor wires onto itself, so application code uses them as `modly.<namespace>.<method>(...)`.

> **Don't edit these files by hand.** They're regenerated by `bot/scripts/generate-sdk.ts` from the live route definitions. To add an endpoint, add the route in the bot and re-run the generator.

## Usage

```ts
import { ModlyClient } from "@modly_public/sdk";

const modly = new ModlyClient({ guildId, apiKey });

// Generated namespaces are accessed identically to hand-rolled ones:
const rules = await modly.automod.listRules();
const leaders = await modly.leveling.listRules();
const appeals = await modly.appeals.list();
```

You can also import an attacher and bind a sub-client:

```ts
import { attachAutomod } from "@modly_public/sdk";
const automod = attachAutomod(myClient);
```

## Adding an endpoint

1. Add the route in the bot (`bot/src/api/routes/<name>.ts`).
2. Run `bot/scripts/generate-sdk.ts` (drives `npm run generate:sdk` in the SDK).
3. Commit the regenerated namespace file alongside the route.
